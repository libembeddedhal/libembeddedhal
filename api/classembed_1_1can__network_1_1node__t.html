Detailed Description

A can network node stores the can messages in a lock free way.

Updating the can message data is completely lock free. Retrieving data uses atomics to poll for when the update() function has completed. This asymmetry in locking is to reduce write time, which is usually done in an interrupt context, rather than than read time, which is performed by a driver in a thread or main thread.
